pullit together with storethat are the two workhorses of the finRes suite. Install the development version with devtools::install_github("bautheac/pullit").
pullit works in tandem with BBGsymbols to provide the R user with easy access to Bloomberg financial data for a collection of financial instruments that at the time of writting includes equity and equity-like securities, funds with the category encompassing any money-managing entity, as well as futures series and individual contracts.
For each instrument category, pullit allows data retrieval for a corresponding category specific set of data types.

Load package & assign global variables:

library(pullit)

start <- Sys.Date() - lubridate::years(2L); end <- Sys.Date()

Bloomberg

An active Bloomberg connection is required to retrieve Bloomberg financial data using pullit.

equity

Assign equity variables:

equity_tickers <- c("ADM US Equity", "KHC US Equity", "XPO US Equity")

Equity data comes in two major categories, market and book. For a given corporation market data records stock market activity while book data records business activity in the form of financial statements commonly refered to as ‘books’. See the ‘fields’ dataset in BBGsymbols for a detailled list of the Bloomberg fields available for data retrieval in pullit.

market

Retrieve market data from Bloomberg for Archer-Daniels-Midland (ADM US Equity), Kraft-Heinz (KHC US Equity) and XPO Logistics (XPO US Equity) with:

equity_market <- BBG_equity_market(equity_tickers, start, end)
#> <U+2714> ADM US Equity
#> <U+2714> KHC US Equity
#> <U+2714> XPO US Equity

This returns a S4 object of class ‘EquityMarket’:

equity_market
#> S4 object of class EquityMarket 
#> 
#> Slots inlude
#>    tickers: access with get_tickers()
#>    fields: access with get_fields()
#>    data: access with get_data()
#>    call: access with get_call()
#>  
#> See also: get_periods()

The object carries a ‘tickers’ dataframe that indicates the tickers for which some data have been found, a ‘fields’ dataframe that indicates the data fields for which data has been found, a ‘data’ dataframe that hosts the retrieved data as well as a character vector hosting the original call to the function. All these can be accessed using the ‘@’ operator or more formaly using accessor methods provided by finRes. A get_periods() method complements the accessor methods by indicating the start and end dates between which data have been found for each ticker and data field.
